# ADR-001: Инструменты для построения и наблюдаемости RAG пайплайна

**Статус:** Принято  
**Дата:** 2026-05-11  
**Автор:** Nikita Babchenko

---

## Контекст

Проект `rag-by-book` — учебный RAG-пайплайн на основе текста книги. Текущий стек: ChromaDB + sentence-transformers + Anthropic Claude Haiku + CLI. Пайплайн работает, но не хватает:

- Понимания, **почему** ответ получился именно таким (какие чанки нашлись, какой скор)
- Инструментов для **оценки качества** retrieval и генерации
- Более удобного UI чем CLI для экспериментов
- Знания о том, что используют в production

Вопрос: какие инструменты выбрать на каждом из этих уровней?

---

## Категории инструментов

### 1. Оркестрация (Frameworks)

Фреймворки, которые собирают RAG-пайплайн из компонентов:

| Инструмент | Описание | Когда брать |
|---|---|---|
| **LangChain** | Самый популярный. Куча абстракций: chains, agents, retriever. | Для изучения паттернов, большая экосистема |
| **LlamaIndex** | Специализирован на data ingestion + retrieval. Индексы, query engines. | Если важен retrieval над документами |
| **Haystack** | Enterprise-ориентированный. Pipelines как граф компонентов. | Production с кастомными компонентами |

**Для учебного проекта:** LlamaIndex — лучший выбор для глубокого понимания RAG. LangChain удобен для быстрого прототипирования.

---

### 2. Трейсинг и наблюдаемость (Observability)

Позволяют видеть, что происходит внутри LLM-вызовов: какой промпт ушёл, какой ответ вернулся, сколько токенов, какие чанки были найдены.

| Инструмент | Описание | Доступность |
|---|---|---|
| **LangSmith** | Официальный трейсер LangChain. Web UI, трейсы, датасеты, evaluations. | Бесплатный tier (до 5k трейсов/мес) |
| **Langfuse** | Open-source альтернатива LangSmith. Self-hosted или cloud. | Бесплатный self-hosted |
| **Phoenix (Arize)** | Локальный UI для трейсинга и оценки. OpenTelemetry-based. | Полностью бесплатный, локальный |
| **Helicone** | Proxy + аналитика для LLM API. Не требует изменения кода. | Freemium |

**Рекомендация для учёбы:** **Phoenix** — запускается локально одной командой, показывает embeddings, retrieval scores, LLM traces в браузере. Не требует аккаунта.

```bash
pip install arize-phoenix
python -m phoenix.server.main &   # открывает localhost:6006
```

---

### 3. Оценка качества (Evaluation / Evals)

Позволяют измерить, насколько хорошо RAG отвечает на вопросы:

| Инструмент | Что измеряет | Описание |
|---|---|---|
| **RAGAS** | Context Recall, Faithfulness, Answer Relevance | Специализирован под RAG, самый популярный |
| **DeepEval** | Hallucination, Answer Relevancy, Contextual Precision | Широкий набор метрик, CLI + CI integration |
| **TruLens** | RAG Triad: context relevance, groundedness, answer relevance | Хорошая визуализация, интеграция с LlamaIndex |

**Рекомендация:** **RAGAS** — де-факто стандарт для RAG evaluation. Достаточно нескольких вопросов с эталонными ответами.

---

### 4. UI для экспериментов

| Инструмент | Описание | Сложность |
|---|---|---|
| **Gradio** | Python-based web UI. Уже используется в проекте (`app.py`). | Минимальная |
| **Streamlit** | Популярный для ML-демо. Удобный state management. | Низкая |
| **Chainlit** | Специально для LLM чатов. Thread history, feedback кнопки. | Низкая |

**Для этого проекта:** Gradio уже подключён. Chainlit — лучший выбор если нужен chat-интерфейс с историей сессий.

---

### 5. Vector DB UI

| Инструмент | Описание |
|---|---|
| **ChromaDB** | Нет встроенного web UI |
| **chroma-ui** (community) | Неофициальный просмотрщик коллекций |
| **Qdrant Web UI** | Если мигрировать с ChromaDB на Qdrant — есть встроенный дашборд |

---

## Решение

Для данного учебного проекта принимается следующий план:

1. **Оставить** текущий стек как есть (ChromaDB, sentence-transformers, Anthropic)
2. **Добавить Phoenix** для локального трейсинга — видеть что происходит при каждом запросе
3. **Добавить RAGAS** для периодической оценки качества retrieval и ответов
4. **Оставить Gradio** как UI — он уже есть, добавить в него отображение retrieved chunks
5. **Не добавлять** оркестрационный фреймворк (LangChain/LlamaIndex) — текущий "ручной" пайплайн лучше для понимания основ

---

## Отклонённые альтернативы

- **LangSmith** — требует аккаунт, избыточно для учебного проекта
- **Переход на Qdrant** — нецелесообразен, ChromaDB достаточен для одной книги
- **Добавить LangChain** — скрывает детали реализации, что плохо для обучения на этом этапе

---

## Последствия

**Положительные:**
- Phoenix даст визуальное понимание как работает каждый шаг пайплайна
- RAGAS покажет реальные метрики качества, а не только субъективную оценку ответов
- Минимальные изменения в существующем коде

**Отрицательные / риски:**
- Phoenix добавляет тяжёлую зависимость (~200MB)
- RAGAS требует LLM-судью для оценки (дополнительные API расходы)
